# NED Data Dictionary

This document describes the attributes of the main tables in the NED database. This is
a reference document; the authoritative definition of the schema is `ned_app/models.py`
and this document is updated occasionally and may lag behind small schema changes.

**Conventions used below:**

- **Required** means the field must be given a non-empty value when adding a record.
  **Optional** means it may be left blank/empty.
- **Auto** means the field is generated by the ingest pipeline — do **not** fill it in
  yourself; any value you supply is ignored or overwritten.
- *String (max N)* is free text up to N characters. *Text* is free text with no length
  limit. *Choice* fields must exactly match one of the listed values (case-sensitive).
- Shared choice lists (used by more than one table) are defined once in
  [Shared choice vocabularies](#shared-choice-vocabularies) at the end.

---

## Reference

Bibliographic records of the published studies and documents from which all data in
the database is drawn. One row per published source (journal paper, report, guideline document). Every
experiment and fragility model must point at a reference.

| Field | Type | Required | Description |
|---|---|---|---|
| `reference_id` | String (max 255), unique | Auto | Unique identifier, derived at ingest as `<token>-<year>` where the token is the `reference_label` if set, otherwise the first author's surname stripped to letters/digits (e.g. `Smith-2020`, `FEMA_P58-2018`, `ArayaLetelier-2019`). Never assign this manually — any supplied value is stripped and re-derived. |
| `reference_label` | String (max 100) | Optional | Overrides the first-author surname as the id token. Use it for institutional authors (e.g. `FEMA_P58`) or to distinguish two references that would otherwise derive the same id (same surname and year) — in the collision case a label is **required**. Rules: letters, digits, and underscores only (no hyphens or spaces); must not start or end with an underscore; must not be a bare number/year. |
| `title` | String (max 255) | Auto | Title of the paper or manuscript. Populated from `csl_data['title']`; not editable. |
| `author` | String (max 255) | Auto | Author display string built from `csl_data['author']` family names: one author → `Smith`; two → `Smith and Jones`; three or more → `Smith et al.`. Not editable. |
| `year` | Integer | Auto | Publication year, taken from `csl_data['issued']['date-parts']`. Not editable. |
| `study_type` | Choice | Required (defaults to `Other`) | Classification of the type of study conducted. One of the [Study type](#study-type) values. |
| `comp_type` | String (max 255) | Optional | The type of component(s) investigated in the study (e.g. "suspended ceilings"). |
| `pdf_saved` | Boolean (`true`/`false`) | Required (defaults to `false`) | Whether a PDF of the reference is saved in the archive repository. |
| `csl_data` | JSON object (CSL-JSON) | Required | Full bibliographic record in [CSL-JSON](https://citeproc-js.readthedocs.io/en/latest/csl-json/markup.html) format. Must contain a non-empty `title`, a non-empty `author` list, and an `issued` field with `date-parts` containing at least a valid positive-integer year. Do **not** include an `id` key — it is stripped at ingest. **Year policy:** when the online-first year and the print-volume year differ, `issued` holds the online-first (first-available) year. |

## Component

The catalog of nonstructural building component types, classified using the
UNIFORMAT II / NISTIR 6389 hierarchy, optionally extended with suffix levels for
finer component types. One row per type of building component. See the README section *Component Subcategorization Hierarchy* for guidance on
categorizing components.

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | String (max 10), primary key | Auto | Compact form of `component_id` with the four UNIFORMAT II / NISTIR levels concatenated (e.g. `B.20.1.1.A` → `B2011.A`). Generated on save; scheduled to be replaced by an integer key in a future update. |
| `component_id` | String (max 20), unique | Required | Dotted UNIFORMAT II / NISTIR component ID, e.g. `A.10.1.1` or `B.20.1.1.A`. Must have at least four levels, and the first four levels must exist in the UNIFORMAT II / NISTIR taxonomy (`ned_app/schemas/nistir_labels.json`). Additional dotted suffix levels may be appended to distinguish component types within a UNIFORMAT II / NISTIR subelement. This is the value that `Experiment.component` and the component–fragility bridge point at. |
| `name` | String (max 255) | Required | Name of the individual type of building component (e.g. "Suspended ceiling, perimeter-fixed"). |
| `major_group` | String (max 255) | Auto | UNIFORMAT II / NISTIR level-1 ID and description (e.g. `A - Substructure`). Populated from the taxonomy; not editable. |
| `group` | String (max 255) | Auto | UNIFORMAT II / NISTIR level-2 ID and description. Not editable. |
| `element` | String (max 255) | Auto | UNIFORMAT II / NISTIR level-3 ID and description. Not editable. |
| `subelement` | String (max 255) | Auto | UNIFORMAT II / NISTIR level-4 ID and description. Not editable. |

## Experiment

Records of observations of performance and damage from experimental testing programs
of nonstructural components. One row per **observation of component damage** (or explicit non-damage) in a single
test of a single specimen. A specimen tested multiple times, or exhibiting multiple
damage states, produces multiple rows.

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | String (max 255), primary key | Required | Unique identifier for this observation, assigned by the contributor. |
| `reference` | Foreign key → `Reference.reference_id` | Required | Derived id of the published reference documenting this observation (e.g. `Smith-2020`). The reference must already exist. |
| `specimen` | String (max 255) | Optional | ID or name of the specimen as recorded in the reference. |
| `specimen_inspection_sequence` | String (max 255) | Optional | Which test of this specimen the observation comes from (the *i*th test), when a specimen was tested/inspected multiple times. |
| `reviewer` | String (max 50) | Optional | Individual or institution responsible for entering this record into the database. |
| `component` | Foreign key → `Component.component_id` | Required | Dotted component ID identifying the component type (e.g. `D.50.2.1.A`). The component must already exist. |
| `comp_detail` | String (max 100) | Optional | Subcategorization tag: classification or short description of the component attachment/connection detailing (e.g. `perimeter-fixed` vs `back-braced`). Use consistent naming, case, and spelling across records. |
| `material` | String (max 100) | Optional | Subcategorization tag: general material grouping of the component, if applicable (e.g. `CPVC` vs `iron` sprinkler pipes). |
| `size_class` | String (max 100) | Optional | Subcategorization tag: general size grouping of this component relative to others of the same type, if applicable. |
| `test_type` | Choice | Required | The loading condition of the test. One of the [Test type](#test-type) values. |
| `loading_protocol` | Text | Optional | Name, ID, or general description of the ground motion or loading protocol used in the test. |
| `peak_test_amplitude` | String (max 255) | Optional | The maximum amplitude to which this test was performed (recorded as text, so units/qualifiers can be included). |
| `location` | String (max 255) | Optional | Where the test was conducted (e.g. lab, institution, and/or country). |
| `governing_design_standard` | String (max 255) | Optional | Name of the standard governing the design of the specimen, if applicable. |
| `design_objective` | Text | Optional | Performance level to which the specimen was designed, e.g. code compliant, common construction practice, low-damage design, or meeting a certain damage objective under a specific loading condition. |
| `comp_description` | Text | Required | General description of the type of component tested. |
| `ds_description` | Text | Required | Description of the damage being observed (or observation of no damage). |
| `prior_damage` | Text | Optional | Description of any damage noted during a *previous* test of this specimen, including if and how the specimen was repaired before this test. Leave empty if no prior damage was noted. |
| `prior_damage_repaired` | Text | Optional | `TRUE` if prior damage was noted and repaired before this test; `FALSE` if noted and not repaired; or a general description of the previous damage that was repaired. |
| `edp_metric` | Choice | Required | The engineering demand parameter (EDP) measured at this observation. One of the [EDP metric](#edp-metric) values. |
| `edp_unit` | Choice | Required | Unit of the EDP. One of the [EDP unit](#edp-unit) values; must be consistent with `edp_metric`. |
| `edp_value` | Decimal (up to 12 digits, 6 decimal places) | Required (blank only when genuinely unavailable) | Value of the EDP recorded for this observation, in `edp_unit`. |
| `alt_edp_metric` | Choice | Optional | Secondary EDP metric, when the reference reports the demand a second way. Same choice list as `edp_metric`. |
| `alt_edp_unit` | Choice | Optional | Unit of the secondary EDP. Same choice list as `edp_unit`. |
| `alt_edp_value` | Decimal (up to 12 digits, 6 decimal places) | Optional | Value of the secondary EDP; stored as null when blank. |
| `ds_rank` | Integer | Optional | Rank ordering this observed damage relative to other damage observed in the same specimen (1 = first/least severe) as recorded by the original author in the reference. Leave blank if not noted in the reference. |
| `ds_class` | Choice | Required | First-pass classification of whether the damage is consequential, assigned by the individual or institution entering this record into the database. One of the [Damage state class](#damage-state-class) values. When in doubt, err toward `Consequential`; use `Unknown` only when a class cannot be identified. See the README section *DS Class* for additional information. |
| `notes` | Text | Optional | Additional notes providing context for the damage observation. |

## Fragility Model

Seismic fragility models describing the sequence of damage of a particular nonstructural component configuration as a function of demand. One row per unique fragility model. The damage states of a given fragility model are defined in the Fragility Curve table.

| Field | Type | Required | Description |
|---|---|---|---|
| `fragility_model_id` | String (max 255), unique | Auto | Generated on save as `<reference_id>\|<model_id>` (e.g. `Smith-2020\|fra001`). This is the value the curve table and bridge tables point at. |
| `reference` | Foreign key → `Reference.reference_id` | Required | Derived id of the reference documenting this fragility model. |
| `model_id` | String (max 255) | Required | Model identifier chosen by the contributor; must be unique within the reference (the pair reference + model_id is enforced unique). Use an id provided by the original author if available in the reference. |
| `p58_fragility` | String (max 50) | Optional | FEMA P-58 fragility id associated with this model, if applicable (e.g. `C3032.001a`). |
| `comp_detail` | String (max 100) | Optional | Subcategorization tag: classification or short description of the component attachment/connection detailing (e.g. `perimeter-fixed` vs `back-braced`). Use consistent naming, case, and spelling across records. |
| `material` | String (max 100) | Optional | Subcategorization tag: general material grouping of the component, if applicable (e.g. `CPVC` vs `iron` sprinkler pipes). |
| `size_class` | String (max 100) | Optional | Subcategorization tag: general size grouping of this component relative to others of the same type, if applicable. |
| `comp_description` | Text | Required | General description of the type of component the model applies to. |
| `reviewer` | String (max 255) | Optional | Person or party responsible for uploading this fragility model to the database. |
| `source` | String (max 255) | Optional | Source of the fragility data. |
| `edp_metric` | Choice | Required (database-enforced non-empty) | The EDP the fragility curves are defined against. One of the [EDP metric](#edp-metric) values. |
| `edp_unit` | Choice | Required (database-enforced non-empty) | Unit of the EDP. One of the [EDP unit](#edp-unit) values. |

## Fragility Curve

The individual lognormal damage-state curves that make up each fragility model. One row per individual fragility curve — a lognormal distribution for one damage state within a fragility model.

| Field | Type | Required | Description |
|---|---|---|---|
| `fragility_model` | Foreign key → `FragilityModel.fragility_model_id` | Required | Id of the fragility model this curve belongs to (e.g. `Smith-2020\|fra001`). |
| `basis` | Choice | Optional | Observational basis of the underlying data comprising the curve. One of the [Study type](#study-type) values. |
| `num_observations` | Integer | Optional | Number of observations that form the basis of the curve. |
| `ds_rank` | Integer | Optional | Rank ordering this curve relative to the other curves in the same fragility model (1 = first/least severe damage state). |
| `ds_description` | Text | Required | Description of the damage being modeled by this curve. |
| `median` | Decimal (up to 9 digits, 4 decimal places) | Required | Median of the lognormal fragility curve, in the model's `edp_unit`. Must be strictly greater than 0. |
| `beta` | Decimal (up to 4 digits, 3 decimal places) | Required | Lognormal dispersion (standard deviation of the log of the demand). Must be strictly greater than 0. |
| `probability` | Decimal (up to 3 digits, 2 decimal places) | Required | Mutually exclusive probability of this damage state, for models with mutually exclusive damage states. Must be between 0 and 1 inclusive. Use a probability of 1 for purely sequential damage states. |

## Shared choice vocabularies

Choice fields must match one of these values **exactly** (including case and
punctuation).

### Study type

Used by `Reference.study_type` and `FragilityCurve.basis`.

| Value | Meaning |
|---|---|
| `Experiment` | Laboratory experimental testing. |
| `Historical Event` | Reconnaissance observations from a real (historical) event. |
| `Analytical Study` | Analytical/numerical study. |
| `Lit Review` | Literature review / compilation of prior studies. |
| `Judgment` | Expert judgment. |
| `Other` | Anything not covered above. |

### Test type

Used by `Experiment.test_type`.

| Value |
|---|
| `Dynamic, uniaxial` |
| `Dynamic, bi-directional` |
| `Dynamic, horizontal and vertical` |
| `Dynamic, 3D` |
| `Monotonic, compression` |
| `Monotonic, tension` |
| `Monotonic, bending` |
| `Monotonic, lateral` |
| `Quasi-static Cyclic, uniaxial` |
| `Quasi-static Cyclic, bi-directional` |

### EDP metric

Used by `Experiment.edp_metric`, `Experiment.alt_edp_metric`,
and `FragilityModel.edp_metric`.

| Value |
|---|
| `Story Drift Ratio` |
| `Story Drift Ratio, bi-directional` |
| `Residual Story Drift Ratio` |
| `Peak Floor Acceleration, horizontal` |
| `Peak Table Acceleration, horizontal` |
| `Peak Floor Acceleration, vertical` |
| `Peak Floor Velocity` |
| `Joint Rotation` |
| `Force, tension` |
| `Force, compression` |
| `Force, bending` |
| `Force, lateral` |
| `Custom` |

### EDP unit

Used by `Experiment.edp_unit`, `Experiment.alt_edp_unit`,
and `FragilityModel.edp_unit`.

| Value |
|---|
| `g` |
| `Ratio` |
| `Radians` |
| `Kips` |
| `k-in` |
| `Meters Per Second` |
| `Custom` |

### Damage state class

Used by `Experiment.ds_class`. See the README section *DS Class* for the full
discussion of intent and edge cases.

| Value | Meaning |
|---|---|
| `No damage` | No change in state was observed from the test. |
| `Inconsequential` | (Aesthetic) damage was observed but is unlikely to require repair or impact system operation — no action required. |
| `Consequential` | Damage that may require repair or impact system operation — observable and requires action. |
| `Unknown` | A damage state class could not be identified by the reviewer. |
